Explainability of the Model

The current model is in beta 0.19. While randomised forest was used toupdate the model, a simple tree was used in the end. This was due to having an average better accuracy for each bean compaired to the random forest.

The Construction of the Model

The dataset used to fit the model was The Dry Bean Data Set created by Murat Koklu and Ali Ozkan. There was a lot of data in this, but the data was formed to reduce possible weights by forcing all the features to be pushed to a number inbetween 0 and 1. There was also more data on certain beans than on others, so the data was changed so that each classification had a certain number of training data that equaled the others.

From here, the tree model was formed to not allow overfitting be setting the min/max node split and leaf. This insured that the model would not fit the training data perfectly as it should look at common features of each bean. From each bean, 11 features were used:Area, Perimeter, MinorAxisLength, AspectRation, Eccentricity, ConvexArea, EquivDiameter, Extent, Solidity, roundness, compactness, shapefactor1, 2, 3, 4.

The Features

By far the most used feature is perimeter. While conexArea, extent, and eccentricity were barley used, they had minor changes that bettered the accuracy slightly for the beans. More features usually mean a better accuracy, but that was not fully the case as MajorAxisLength was removed. It can also be shown that the graph is exponential. With each feature added, it seemed that thenext would mean exponentially less to the prediction.
